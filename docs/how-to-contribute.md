# How To Contribute

## Goal

Add new resources using a consistent structure so the repo stays:

- Easy to search
- Easy to compare
- Free from duplication and low-value notes

## Process For Adding One Link

1. Identify the resource type:
   - Repo
   - Official docs
   - Paper
   - Benchmark
   - Tutorial
   - Tool
2. Identify the main topic area:
   - AI foundations
   - AI agents
   - .NET + AI
   - Ecosystem
3. Choose the matching file under `lists/`.
4. Copy the template from [../templates/entry-template.md](../templates/entry-template.md).
5. Fill in the required fields.
6. Place the entry in a sensible location in the file:
   - Group by topic when needed
   - Within a group, sort by importance or alphabetically

## When Not To Add A Link

- The link is too generic and has no clear reference value
- The project is archived and no longer useful for history or comparison
- The link is only a mirror, re-upload, or unofficial copy
- The resource is mostly marketing rather than technical substance

## Quality Rules

- `Why it matters` should answer: "Why should I care about this resource?"
- `Compare with` should help readers branch into adjacent references when relevant
- `Notes` should stay short and useful

## If No Category Fits

Read [tagging-guide.md](./tagging-guide.md) first.

Only create a new category when:

- A clear topic cluster is emerging
- The current category is becoming too broad
- There are at least 3 valid resources for the new category
